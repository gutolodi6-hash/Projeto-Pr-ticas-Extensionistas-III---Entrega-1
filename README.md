GESTÃO DE PEDIDOS E ATENDIMENTO: MODELAGEM DE SISTEMA DE INFORMAÇÃO E BANCO DE DADOS
Autor: Sérgio Augusto Lodi

Introdução: As novas tecnologias revolucionaram a jornada de consumo, obrigando as organizações a se adequarem a um novo público que busca agilidade e eficiência no momento da compra. No contexto de sistemas de informação, a modelagem precisa e estruturada é o alicerce para que soluções digitais entreguem valor real, permitindo uma comunicação fluida entre os processos de negócio e a tecnologia. Assim como o marketing digital busca atrair e reter clientes, um sistema de gestão bem modelado visa organizar o fluxo de dados para otimizar a operação e melhorar a experiência do usuário.

Objetivo: O presente trabalho tem por objetivo desenvolver a modelagem completa do sistema de informação e do banco de dados para um software de Gestão de Pedidos e Atendimento, identificando as entidades, relacionamentos e fluxos operacionais necessários para a automação do processo de vendas.

Metodologia: Quanto à natureza e objetivos, este estudo classifica-se como uma pesquisa aplicada de desenvolvimento técnico, focada na engenharia de software. Utilizou-se a abordagem qualitativa para a definição de requisitos funcionais; quanto aos procedimentos, realizou-se a modelagem de banco de dados (Conceitual e Lógico) através da ferramenta brModelo, e a modelagem UML (Diagramas de Caso de Uso, Classes, Sequência e Atividades) por meio de ferramentas como Visual Paradigm e Draw.io. A revisão documental das regras de negócio de um sistema de vendas permitiu a estruturação do funil de dados e da jornada operacional.

Resultados: Identifica-se que a modelagem permite alinhar a estrutura do banco de dados com a persona do operador (funcionário) e do consumidor final, fortalecendo a integridade das informações em cada etapa do pedido. A estratégia de modelagem por diagramas de atividades e sequência mostrou-se eficiente para mapear o "funil" do atendimento, desde o contato inicial até a conversão em venda. Contudo, observou-se que a padronização de fluxos para diferentes perfis de clientes exige uma jornada de sistema bem definida. Constatou-se que a correta aplicação do Diagrama de Classes, em conjunto com o Modelo Lógico de dados, valida o processo de compra do ponto de vista sistêmico, identificando pontos de contato cruciais para a retenção de dados e suporte ao cliente.

Conclusão: Com base no presente estudo, foi possível reconhecer que a modelagem prévia fortalece o relacionamento entre os requisitos do negócio e a implementação tecnológica. A utilização de diagramas UML no desenho do sistema é eficaz para prever falhas e otimizar a experiência de uso. Sugere-se, para etapas futuras, o desenvolvimento do protótipo de interface para validar o comportamento do usuário antes, durante e após a experiência de compra no sistema.

Palavras-chave: Modelagem de Dados. UML. Gestão de Pedidos. Banco de Dados. Engenharia de Software.
